## Instill Cloud

Try [Instill Cloud](https://console.instill.tech/?utm_source=documentation) for the easiest and most reliable experience to access all the fantastic features of Instill Core.

## Self-host

You have the option to self-host locally using Instill CLI, Docker Compose, Kubernetes with Helm, and more.
We are continually optimizing the self-hosting process to get you up and running with our services in just a few minutes.
If you encounter any challenges with self-deployment, please don't hesitate to reach out to us on [GitHub](https://github.com/instill-ai/community/issues/new/choose) or join our [Discord](https://discord.gg/sevxWsqpGh) community for assistance.

### Deploy with Instill CLI

Please checkout [Instill CLI](/docs/cli). This is the recommended way to access both Instill Core and Instill Cloud.
For the best experience accessing both Instill Core and Instill Cloud, we recommend using the [Instill CLI](/docs/cli).

### Deploy with other methods

Alternatively, you can deploy Instill Core via:

- [Docker Compose](docker-compose)
- [Kubernetes using Helm](kubernetes-using-helm)
